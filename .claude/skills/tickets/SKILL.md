---
name: tickets
description: works with the user to define a backlog of tickets based on known delivery milestones
---

Work with the user to process the $ARGUMENTS into individual tickets.

Each ticket should have its own file. Each file should be named based upon the milestone and ticket. So a ticket for milestone 1 ticket 1 would be '1.1-do-a-thing.md'.

Each ticket should deliver a vertical slice of functionality. The vertical slice should strive to be as thin as possible whilst still providing value. So "create a database" would be a bad ticket but "add a user to the system" would be a good one.

Work with the user iteratively, milestone by milestone and ask them questions to identify the correct slices.
