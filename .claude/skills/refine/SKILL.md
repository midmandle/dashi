---
name: refine
description: work with a user to refine a ticket
---

Work iteratively to refine the ticket defined in $ARGUMENTS.

Each ticket should have:

- a high-level description of the work, including an expected outcome.
- a user story in the format "As a USER... I want to be able to... So that I can..."
- a list of acceptance criteria
- a E2E test which defines the steps to test the ticket end-to-end
- a list of tests which exercise the acceptance criteria
- any additional technical information to aid with delivery

Work with the user incrementally. Ask them questions where needed. Ensure that they approve the contents of the ticket before editing the file.
