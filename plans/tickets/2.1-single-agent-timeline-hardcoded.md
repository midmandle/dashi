# 2.1 — Single-agent timeline with hardcoded data

Build the React + Vite app shell with a layout designed to accommodate multiple agents in Phase 3. Includes a timeline view for a single agent that fetches events from a real `GET /events` Lambda endpoint and renders them as a simple ordered list. The Lambda returns hardcoded fixture data for this ticket — DynamoDB integration comes in a later ticket. No streaming — events are fetched once on load.

**Expected outcome:** A running React app that displays a simple list of events for a single agent, fetched from a real deployed endpoint returning hardcoded data, with a layout ready to extend to multiple agents.

---

**User story**
As a developer, I want to see a single-agent event list rendered in a browser so that I can validate the UI and endpoint structure before connecting real data.

---

**Acceptance criteria**
1. A React + Vite app exists and renders in the browser
2. A `GET /events` Lambda endpoint exists and returns a hardcoded list of events
3. The Lambda and API Gateway route are provisioned via Terraform
4. The API endpoint URL is configurable in the React app via an environment variable
5. The app fetches events from the endpoint on load and displays them as a list ordered by timestamp
6. The layout is structured to accommodate multiple agents in a future ticket
7. The app is served locally via a dev server

---

**Acceptance tests**
- `GET /events` returns a `200` with a hardcoded list of events
- App loads in the browser without errors
- Timeline renders the correct number of events returned by the endpoint
- Events are displayed in chronological order
- Each event displays at minimum its `type` and `timestamp`

---

**Technical notes**
- `GET /events` Lambda returns hardcoded fixture data — no DynamoDB reads in this ticket
- API endpoint URL injected into the React app via environment variable (`VITE_API_URL`)
- No streaming, polling, or real-time updates in this ticket
- Layout should anticipate a multi-agent view (e.g. one panel per agent) even if only one is rendered here
