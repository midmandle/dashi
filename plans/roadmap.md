# Roadmap — Dashi MVP

## Phase 1 — Agent Status Updates

Agents can publish status updates to Dashi. This is the core primitive everything else builds on — without it, there is nothing to display.

**Capability:** Any agent, from anywhere on the internet, can post event updates to Dashi.

---

## Phase 2 — Live Agent Progress

A user can open Dashi and watch a single agent's work unfold in real time. Validates the display model before expanding to multiple agents.

**Capability:** A user can see a single agent's progress as it happens — without reading logs or chasing updates.

---

## Phase 3 — Parallel Agent Monitoring

The full use case: a senior engineer running multiple agents in parallel can monitor all of them from a single screen.

**Capability:** A user can observe the live progress of multiple concurrent agents on one screen, with no performance degradation as agent count grows.
